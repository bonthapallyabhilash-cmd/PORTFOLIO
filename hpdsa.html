Java Hospital Management System

A console-based Hospital Management System developed using Java. This project demonstrates the implementation of core Data Structures such as Linked Lists, Stacks, Queues, and Arrays to efficiently manage hospital operations including patient records, doctor management, and visit history.

🚀 Features

. Patient Waiting Queue (Queue): Patients are added to a waiting queue and admitted based on the First-In First-Out (FIFO) principle.
. Admitted Patients List (Linked List): Stores and manages the list of admitted patients, allowing insertion and deletion operations dynamically.
. Patient Discharge: Enables deletion of patient records when a patient is discharged from the hospital.
. Doctor Management (Array / Linked List): Add, delete, search, and display doctors available in the hospital.
. Patient Visit History (Stack): Maintains a record of recently visited patients using Last-In First-Out (LIFO) structure.
. Prescription Validation (Stack Application): Uses stack logic to validate prescriptions or medical records format.

🛠️ Technical Implementation

. This project focuses on manual implementation of data structures without relying heavily on Java’s built-in collection framework.
. Custom Node Structure: Created a HospitalRecord class to store patient and doctor details.
. Pointer Management: Used next and prev references to manage a doubly linked list for efficient record handling.
. Queue Operations: Implemented enqueue and dequeue logic to manage patient admission order.
. Stack Operations: Used stack principles for visit history and prescription validation.
. Search Operations: Implemented linear search to locate doctors or patients quickly.
